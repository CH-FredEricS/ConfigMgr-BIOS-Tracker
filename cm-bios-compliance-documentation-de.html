ConfigMgr BIOS Compliance Checker

Secure Boot 2023 · UEFI-Zertifikats-Voraussetzungen · v1.10

01Einleitung

Der ConfigMgr BIOS Compliance Checker ist ein eigenständiges HTML-Tool, das Geräte in Ihrer Microsoft Endpoint Configuration Manager-Umgebung (MECM / SCCM) identifiziert, deren BIOS-Version nicht den Mindestanforderungen für das Secure Boot 2023 UEFI-Zertifikats-Update entspricht.

Geräte, die das Windows Update mit dem neuen Secure-Boot-Zertifikat erhalten, ohne über ein ausreichend aktuelles BIOS zu verfügen, können einen Boot-Fehler erleiden. Die Hersteller haben Mindest-BIOS-Versionen veröffentlicht; dieses Tool prüft Ihr gesamtes ConfigMgr-Inventar in Sekunden gegen diese Anforderungen. Virtuelle Maschinen werden automatisch ausgeschlossen — sie können keine BIOS-Updates erhalten.

ℹ️
Keine serverseitige Komponente erforderlich Das Tool läuft vollständig im Browser. Es liest die aus ConfigMgr exportierten Daten — kein Agent, kein Backend, keine Netzwerkverbindung zur Laufzeit erforderlich.

Funktionsweise

  1. Hardware-Inventardaten (BIOS-Version, Modell, Hersteller) aus ConfigMgr per PowerShell-Skript oder SSRS-Bericht exportieren.
  2. Die exportierte .json- oder .csv-Datei in das Tool laden.
  3. Das Tool gleicht jedes Gerät gegen die integrierte BIOS-Datenbank ab und vergleicht die installierte Version mit dem Herstellerminimum. Virtuelle Maschinen werden automatisch ausgeschlossen.
  4. Die Ergebnisse werden in einer filterbare, sortierbare Tabelle mit Übersichtskacheln angezeigt. Nicht-konforme Geräte werden standardmäßig oben sortiert.

03Schritt 1 — Gerätedaten exportieren

Das Tool akzeptiert zwei Dateiformate: JSON (aus dem PowerShell-Exportskript) und CSV (aus SSRS oder einem anderen tabellarischen Export). Beide enthalten dieselben Kernfelder.

Option A — PowerShell-Export (empfohlen)

Führen Sie das folgende Skript auf einem Rechner mit Netzwerkzugriff auf den ConfigMgr-Datenbankserver aus. Es fragt die Hardware-Inventar-Views direkt ab und schreibt eine JSON-Datei. Das Skript erkennt automatisch, ob Win32_ComputerSystemProduct im Hardware-Inventar aktiviert ist, und fügt das CSP-Versionsfeld bei Verfügbarkeit hinzu.

# ConfigMgr BIOS-Exportskript — v1.10
# Voraussetzung: Read-Only Analyst-Rolle oder gleichwertige SQL-Leseberechtigungen

param(
  [string]$SiteServer  = "CMSERVER",
  [string]$SiteCode    = "P01",
  [string]$OutputFile  = ".\cm_bios_export.json"
)

$conn = New-Object System.Data.SqlClient.SqlConnection
$conn.ConnectionString = "Server=$SiteServer;Database=CM_$SiteCode;" +
                         "Integrated Security=SSPI;Connect Timeout=30"
$conn.Open()

# Prüfen, ob Win32_ComputerSystemProduct im Hardware-Inventar aktiviert ist.
# Version0 enthält den SMBIOS-Typ-1-String. Bei HP-Geräten enthält dieser
# "SBKPFV3", wenn das BIOS die neuen Secure-Boot-Zertifikate bereits unterstützt.
$chkCmd = $conn.CreateCommand()
$chkCmd.CommandText = "SELECT COUNT(*) FROM sys.views " +
                      "WHERE name = 'v_GS_COMPUTER_SYSTEM_PRODUCT'"
$hasCsp = [int]$chkCmd.ExecuteScalar() -gt 0

if ($hasCsp) {
  Write-Host "Win32_ComputerSystemProduct gefunden — CSPVersion wird einbezogen." -ForegroundColor Cyan
} else {
  Write-Host "Win32_ComputerSystemProduct nicht gefunden — CSPVersion bleibt leer." -ForegroundColor Yellow
}

$cspSelect = if ($hasCsp) { ",`n  csp.Version0 AS CSPVersion" } else { ",`n  NULL AS CSPVersion" }
$cspJoin   = if ($hasCsp) { "`nLEFT JOIN v_GS_COMPUTER_SYSTEM_PRODUCT csp ON sys.ResourceID = csp.ResourceID" } else { "" }

$query = @"
SELECT DISTINCT
  sys.Name0               AS ComputerName,
  cs.Manufacturer0        AS Manufacturer,
  cs.Model0               AS Model,
  bios.SMBIOSBIOSVersion0 AS BIOSVersion,
  bios.ReleaseDate0       AS BIOSReleaseDate,
  hw.LastHWScan           AS LastHardwareScan$cspSelect
FROM v_R_System sys
JOIN v_GS_COMPUTER_SYSTEM    cs   ON sys.ResourceID = cs.ResourceID
JOIN v_GS_PC_BIOS            bios ON sys.ResourceID = bios.ResourceID
LEFT JOIN v_GS_WORKSTATION_STATUS hw ON sys.ResourceID = hw.ResourceID$cspJoin
WHERE sys.Client0 = 1
  AND sys.Operating_System_Name_and0 LIKE '%Windows%'
ORDER BY sys.Name0
"@

$cmd = $conn.CreateCommand(); $cmd.CommandText = $query; $cmd.CommandTimeout = 120
$reader = $cmd.ExecuteReader(); $results = @()
while ($reader.Read()) {
  $results += [PSCustomObject]@{
    ComputerName     = $reader["ComputerName"]
    Manufacturer     = $reader["Manufacturer"]
    Model            = $reader["Model"]
    BIOSVersion      = $reader["BIOSVersion"]
    BIOSReleaseDate  = if ($reader["BIOSReleaseDate"] -is [DBNull]) { $null }
                       else { $reader["BIOSReleaseDate"].ToString("yyyy-MM-dd") }
    LastHardwareScan = if ($reader["LastHardwareScan"] -is [DBNull]) { $null }
                       else { $reader["LastHardwareScan"].ToString("yyyy-MM-dd HH:mm") }
    CSPVersion       = if ($reader["CSPVersion"] -is [DBNull]) { $null }
                       else { $reader["CSPVersion"] }
  }
}
$reader.Close(); $conn.Close()
$results | ConvertTo-Json -Depth 3 | Out-File $OutputFile -Encoding UTF8
Write-Host "Exportiert: $($results.Count) Geräte nach $OutputFile" -ForegroundColor Green

Skript-Parameter

ParameterStandardBeschreibung
-SiteServerCMSERVERHostname oder IP des SQL Servers mit der ConfigMgr-Datenbank.
-SiteCodeP01Dreistelliger ConfigMgr-Standortcode. Datenbankname wird als CM_<SiteCode> abgeleitet.
-OutputFile.\cm_bios_export.jsonPfad und Dateiname der JSON-Ausgabedatei.

CSP-Version — HP SBKPFV3-Bereitschaft (optional)

Für HP-Geräte haben Microsoft und HP eine Mindest-BIOS-Version als Voraussetzung für die sicheren Installation der neuen Secure-Boot-Zertifikate definiert. HP veröffentlicht einen zusätzlichen Bereitschaftsindikator im SMBIOS-Typ-1-Versionsfeld: Ist die Zeichenfolge SBKPFV3 vorhanden, unterstützt das BIOS die neuen Zertifikate bereits und es ist kein BIOS-Update vor dem Zertifikat-Rollout nötig.

Dieses Feld wird über Win32_ComputerSystemProduct.Version in WMI bereitgestellt, was v_GS_COMPUTER_SYSTEM_PRODUCT.Version0 in ConfigMgr entspricht. Das PowerShell-Skript erkennt automatisch, ob diese Inventarklasse aktiviert ist, und bindet sie bei Vorhandensein ein.

ℹ️
Win32_ComputerSystemProduct im Hardware-Inventar aktivieren In der ConfigMgr-Konsole: Verwaltung → Clienteinstellungen → [Ihre Einstellungen] → Hardwareinventur → Klassen festlegen → Hinzufügen → Win32_ComputerSystemProduct. Nach dem nächsten Hardware-Inventar-Zyklus wird das CSP-Versionsfeld befüllt.

Manuelle Prüfung auf einem HP-Gerät mit PowerShell:

(Get-CimInstance -ClassName Win32_ComputerSystemProduct).Version
# Beispielausgabe wenn bereit:    "HP ... SBKPFV3 ..."
# Beispielausgabe wenn nicht bereit: "HP EliteDesk 800 G6"  (kein SBKPFV3)

Quelle: HP ISH_13070353 — HP Business PCs: Prepare for new Windows Secure Boot certificates.

Option B — SSRS-Export

Erstellen Sie einen benutzerdefinierten Bericht im ConfigMgr Reporting Services-Portal mit der nachstehenden Abfrage und exportieren Sie die Ergebnisse als CSV (durch Trennzeichen getrennt). Der CSP-Versions-JOIN ist als optionaler Kommentar enthalten — entfernen Sie ihn, wenn Win32_ComputerSystemProduct in Ihrer Umgebung nicht aktiviert ist.

-- SSRS-Berichtsdatensatz-Abfrage (v1.10)
-- CSPVersion-Zeile und JOIN entfernen, falls Win32_ComputerSystemProduct nicht aktiviert.
SELECT DISTINCT
  sys.Name0                                    AS ComputerName,
  cs.Manufacturer0                             AS Manufacturer,
  cs.Model0                                    AS Model,
  bios.SMBIOSBIOSVersion0                      AS BIOSVersion,
  CONVERT(varchar(10), bios.ReleaseDate0, 120) AS BIOSReleaseDate,
  CONVERT(varchar(16), hw.LastHWScan, 120)     AS LastHardwareScan,
  ISNULL(csp.Version0, '')                     AS CSPVersion
FROM v_R_System sys
  JOIN v_GS_COMPUTER_SYSTEM    cs   ON sys.ResourceID = cs.ResourceID
  JOIN v_GS_PC_BIOS            bios ON sys.ResourceID = bios.ResourceID
  LEFT JOIN v_GS_WORKSTATION_STATUS          hw  ON sys.ResourceID = hw.ResourceID
  LEFT JOIN v_GS_COMPUTER_SYSTEM_PRODUCT     csp ON sys.ResourceID = csp.ResourceID
WHERE sys.Client0 = 1
  AND sys.Operating_System_Name_and0 LIKE '%Windows%'
ORDER BY sys.Name0
⚠️
SSRS JSON-Export wird nicht unterstützt Bitte CSV (durch Trennzeichen getrennt) verwenden. SSRS-Datenfeed-Exporte sind im ATOM-Format, nicht JSON.

Im ConfigMgr Reporting Services-Portal den Secure Boot 2023-Konformitätsbericht suchen und als CSV (durch Trennzeichen getrennt) exportieren. Erwartete Spalten:

SpalteBeispielwerte

Bericht laden

  1. Die exportierte CSV auf die Ablagefläche in Schritt 3 ziehen oder auswählen.
  2. Das Tool erkennt Computer_Name und Secure_Boot_2023_Status automatisch. Ggf. manuell anpassen.
  3. Auf Anwenden klicken. Geräte werden nach Computername (Groß-/Kleinschreibung wird ignoriert) abgeglichen.

Abgleichergebnisse

Nach dem Anwenden enthält die Ergebnistabelle eine Secure Boot-Spalte und die Filterleiste einen Secure Boot-Dropdown. Zwei neue Übersichtskacheln erscheinen:

Die Datenbank ordnet Hersteller + Modellname der Mindest-BIOS-Version zu, die vor der sicheren Anwendung des Secure-Boot-2023-UEFI-Zertifikatsupdates erforderlich ist. Zusätzlich enthält die LENOVO_MT-Nachschlagtabelle ~250 Lenovo-4-stellige Maschinentyp-Codes mit den zugehörigen Datenbankeinträgen.

⚠️
Aktualität der Datenbank Hersteller aktualisieren ihre Anforderungen kontinuierlich. Bitte kritische Entscheidungen direkt gegen die oben verlinkten Hersteller-KB-Artikel prüfen.

Modell-Erkennung

ConfigMgr-Modellstrings enthalten oft zusätzliche Wörter, die nicht in der Herstellerdatenbank vorhanden sind. Das Tool verwendet eine vierstufige Erkennungs-Pipeline:

Stufe 1
Exakte Übereinstimmung (Groß-/Kleinschreibung ignoriert)
Beide Strings werden normalisiert (Kleinbuchstaben, Satzzeichen → Leerzeichen) und verglichen.
↓ bei keiner Übereinstimmung
Stufe 2
Teilstring-Enthaltensein — längster gewinnt
Wenn der Gerätename den DB-Eintrag als Teilstring enthält (oder umgekehrt), wird der längste Eintrag verwendet. Behandelt Suffixe wie "Notebook PC".
↓ bei keiner Übereinstimmung
Stufe 3
Jaccard-Wort-Ähnlichkeit ≥ 60 %
Die Wortmengen beider Strings werden verglichen. Schnittmenge/Vereinigung ≥ 0,60 → bester DB-Eintrag wird verwendet.
↓ bei keiner Übereinstimmung (nur Lenovo)
Stufe 4
Lenovo Maschinentyp-Code-Fallback
Wenn alle namensbasierten Stufen für ein Lenovo-Gerät fehlschlagen, extrahiert das Tool die ersten 4 Zeichen des Modellstrings als Maschinentyp-Code und sucht diesen in der LENOVO_MT-Tabelle. Siehe Lenovo MT-Nachschlag.

Lenovo Maschinentyp-Code-Nachschlag

ConfigMgr meldet normalerweise Lenovo-Produktnamen über Win32_ComputerSystem.Model (z. B. ThinkPad T14 Gen 1), die die dreistufige Namenssuche gut verarbeitet. Einige Lenovo-Geräte — typischerweise solche, bei denen das BIOS nach dem Werk nie aktualisiert wurde — melden jedoch nur ihren 4-stelligen Maschinentyp-Code (z. B. 20S1) in diesem Feld.

Die LENOVO_MT-Tabelle ordnet ~250 Maschinentyp-Codes dem entsprechenden BIOS-Datenbankschlüssel zu. Wenn die Stufen 1–3 für ein Lenovo-Gerät fehlschlagen, extrahiert Stufe 4 die ersten 4 Zeichen des Modellstrings und sucht darin.

BIOS-Familien-Disambiguierung

Einige Lenovo-Maschinentypen haben mehr als eine BIOS-Plattform mit unterschiedlichen Mindestversionen. Beispielsweise wurde das ThinkPad P14s Gen 2 mit der N34ET- oder N3WET-BIOS-Plattform ausgeliefert (unterschiedliche Minima: 1,61 vs. 1,13). Die LENOVO_MT_BIOS-Tabelle verwendet die ersten 5 Zeichen des BIOS-Strings als Zusatzschlüssel.

Bei einer Stufe-4-Übereinstimmung über die MT-Tabelle zeigt die Spalte DB-Modell den aufgelösten Produktnamen gefolgt vom MT-Code in Klammern, z. B.: t14 gen 1 (MT:20S1).

Versionsvergleich

Versionsstrings werden an . und - aufgeteilt und jedes Segment numerisch verglichen. 1.9.0 wird korrekt unter 1.10.0 eingeordnet. Wenn keine Version ermittelt werden kann, ist das Ergebnis Nicht geprüft.

09VM-Ausschluss

Virtuelle Maschinen werden vor der Analyse automatisch ausgeschlossen. Sie können keine BIOS-Updates über Standard-Herstellerkanäle erhalten und würden andernfalls falsch-positive Ergebnisse erzeugen. Ausgeschlossene Geräte erscheinen nicht in der Ergebnistabelle, den Compliance-Zählungen oder Exporten.

Wenn mindestens eine VM erkannt wird, erscheint die Kachel VMs ausgeschlossen mit der Anzahl und dem Untertext "nicht analysiert".
